## 1. 平台架構概覽

### 技術棧

- **後端**：Python / Flask (`app.py`, `data_service.py`)
- **前端**：Vanilla JS + Plotly.js（`templates/index.html`, `static/`）
- **研究報告**：R / RMarkdown → 輸出 HTML，放置於 `private/` 資料夾，透過導覽列下拉選單連結

### 現有功能（已完成）

- 大盤監控區：成交量位階儀表板、漲跌家數比例趨勢、委買委賣力道比、成交金額趨勢、熱圖
- 國際股市區：國際指數走勢、匯率換算累積報酬

### 1.1 大盤監控區：特徵溢酬監控（規格定案）

**狀態**：已實作於大盤監控區（`templates/index.html`、`static/js/charts.js`、`data_service.get_feature_premium_*`、`/api/feature-premium/*`）。

**目的**：以統計圖呈現在不同**控制維度（第一維）**下，某一**特徵（第二維）**高相對低組之溢酬隨時間變化，並可選對照加權指數（Y9999）之超額報酬。

**指標**

- **第一維**：依使用者所選變數，於每個再平衡截點將當期橫斷面樣本以**中位數二分**為兩組（高／低）。
- **第二維（主分析變數）**：於上述**每一組內**，依第二維特徵切 **tercile（P1／P2／P3）**；在有效樣本上 **LS\_sub = 高組（P3）等權均值(\(R_i-R_f\)) − 低組（P1）等權均值(\(R_i-R_f\))**。
- **Tercile long–short（主曲線）**：**LS = (LS\_高第一維組 + LS\_低第一維組) / 2**（兩個第一維子樣本等權）。兩子樣本任一侧若該月無法有效切 tercile（檔數不足），該月可僅用另一側或該點留空，規則由實作註記於 README。
- **再平衡**：每個日曆月**最後一個台股交易日**收盤後決定分組，**次一交易日**起適用新權重至下次再平衡。
- **可選對照**：**\(R_{\text{Y9999}} - R_f\)** 之同期表現（資料來源與個股相同管線，證券代碼 **Y9999**）。

**可選欄位**（來自 `因子特徵與載荷.csv` 之特徵欄，不含 `beta_*`）  
`規模`、`淨值市價比`、`益本比`、`股利殖利率`、`動能`、`短期反轉`。

**UI**

- 兩個下拉：**第一維**、**第二維**；第二維選項必須排除與第一維相同之欄位。
- 主圖**同一時間僅顯示一組（第一維 × 第二維）**之 LS 序列；不預設疊加多特徵曲線。

**與日頻因子 CSV**

- `因子報酬.csv`：腳本既定之日頻因子投組報酬。
- 本監控：月再平衡、使用者自選雙維、tercile 等權 LS；頁面須註明差異以免使用者對照數值時誤解。

**資料品質**

- 缺值、樣本不足之交易日或再平衡日：列不入組或該期不繪點（門檻可與 `generate_factor_returns_csv.py` 類似精神，如各 tercile 最少檔數）；避免前視偏差（特徵時點與再平衡時點須與資料字典一致）。
- 統計分布連結（`/stats`）
- 重新載入按鈕（`/api/reload`）

## 3. 新功能一：個股監控頁籤

### 3.1 輸入介面

- 文字輸入框：使用者輸入以逗號隔開的證券代碼（可輸入代碼或代碼+名稱）
  - 範例輸入：`2330, 2303, 2317`
- 查詢按鈕
- 時間範圍選擇：預設近 1 年，按鈕可切換 3 月 / 6 月 / 1 年；支援自訂起訖日

### 3.2 多股比較模式（使用者可切換）

| 模式 | 說明 |
|---|---|
| 疊加圖 | 所有個股以指數化報酬（起始基準 = 100）疊加於同一張圖 |
| 各自顯示 | 每檔股票各自顯示獨立子圖（subplots） |

### 3.3 圖表設計——混頻組合圖

適用於單一股票的詳細頁面，每張圖為「組合圖（dual-axis chart）」：

```
主座標軸（左，深色實線折線圖）：日資料（每交易日有值）
副座標軸（右，淡色面積折線圖）：月/季資料（僅期末有點，點與點直線相連）
```

- X 軸：日曆日期（交易日）
- 月資料的資料點：繪製在各月最後一個交易日
- 季資料的資料點：繪製在各季最後一個交易日
- 空缺的日期（非期末）：月/季線自動以直線補齊

### 3.4 顯示模組

| 模組 | 資料來源 | 主軸 | 副軸 |
|---|---|---|---|
| 股價走勢 | 日資料 | 收盤價（日） | 每股盈餘或選定季財務指標（季） |
| 籌碼 | 籌碼資料庫（日） | 外資/投信/自營買賣超（日） | — |
| 董監持股 | 月資料 | 收盤價（日） | 董監持股%（月） |
| 基本面指標 | 日資料 + 季資料 | 本益比或股淨比（日） | 使用者選定的季財務指標 |

### 3.5 季資料選擇器

- 季資料的副座標軸變數由使用者從下拉選單選擇（選項動態從 CSV 標頭讀取，即完整 200+ 欄）

### 3.6 統計表格

同一頁面顯示統計表格，分為三個 section：

| Section | 說明 |
|---|---|
| 日資料區 | 最新交易日的各指標數值 |
| 月資料區 | 最新月份月底的值（標示月份，例：2026/02） |
| 季資料區 | 最新季度的值（標示季度，例：2025Q3） |

### 3.7 重新載入

與大盤監控區、國際股市區相同，提供「↺ 重新載入」按鈕，觸發後重新讀取 CSV 並刷新頁面資料。

## 4. 新功能二：事件研究 CSV 產生機制

### 4.1 事件類型定義

| 事件名稱 | 判斷條件 | 來源欄位 |
|---|---|---|
| 注意股票(A) | `注意股票(A)` 欄值 = `"A"` | TEJ 股價資料庫 |
| 處置股票(D) | `處置股票(D)` 欄值 = `"D"` | TEJ 股價資料庫 |
| 全額交割(Y) | `全額交割(Y)` 欄值 = `"Y"` | TEJ 股價資料庫 |
| 振幅變大 | 今日 `高低價差%` > 前20交易日 `高低價差%` 之 rolling 第90百分位數 | TEJ 股價資料庫 |

> **rolling 百分位數的計算方式**：以當日前20個交易日（不含當日）的 `高低價差%` 數列，計算其第90/第10百分位數。需至少有20日完整資料，否則該日不產生事件。

| 欄位名稱 | 型態 | 說明 |
|---|---|---|
| 證券代碼 | string | 例：`2330` |
| 年月日 | int | 例：`20250102` |
| 注意股票_A | 0/1 | 1 = 當日為注意股票 |
| 處置股票_D | 0/1 | 1 = 當日為處置股票 |
| 全額交割_Y | 0/1 | 1 = 當日為全額交割 |
| 振幅變大 | 0/1 | 1 = 振幅大事件 |
| 振幅變小 | 0/1 | 1 = 振幅小事件 |

**儲存路徑**：`All_Data/事件資料/事件研究彙整.csv`

### 4.3 更新機制

- 在平台頁面提供「重新產生事件研究CSV」按鈕，點按後觸發 Python 腳本重新掃描所有股價 CSV，輸出最新版的 `事件研究彙整.csv`

## 5. 研究報告 RMarkdown 規格

### 5.0 共用篩選規則

**以下規則適用於所有 RMarkdown 議題（a–i）**：

1. **排除全額交割股**：任何曾被標記為 `全額交割(Y) = "Y"` 的股票，從資料中完全移除
2. **排除處置股票期間**：當 `處置股票(D) = "D"` 時，該股票在**該期間的所有紀錄**移除（非永久排除）
3. **注意股票分組探討**（非排除）：將 `注意股票(A) = "A"` 期間與非注意股票期間**分別**分析，並呈現兩組結果對比
4. **排除上市/上櫃未滿一年的個股**：依 `上市櫃股票清單.csv` 中的上市日期，計算資料截止日時是否滿 252 個交易日
5. **排除成交量為零的紀錄**：`成交量(千股)` = 0 的紀錄直接刪除
6. **除權息期間**（待確認）：若未來有除權息日期資料，排除各股除權息日前後 3 個交易日

**輸出格式**：`output: html_document`，存放於 `private/` 資料夾

**連結方式**：從平台導覽列「研究報告▼」下拉選單以 `target="_blank"` 開啟

### 5.1 議題 a：振幅分析

**研究問題**：個股振幅異常時（大或小），後續報酬、振幅、成交量有何特徵？

#### 事件定義

| 事件 | 條件 |
|---|---|
| 振幅變大 | 今日 `高低價差%` > 前20日 rolling 第90百分位數 |
| 振幅變小 | 今日 `高低價差%` < 前20日 rolling 第10百分位數 |

使用 `高低價差%` 欄位（即 (最高−最低)÷前日收盤×100），不重新計算。

#### 個股層級分析

**後續振幅分析（方案 B）**：

```
異常振幅(T+k) = 高低價差%(T+k) / mean(高低價差%, T-20 to T-1) - 1
```

- 若 `異常振幅(T+k)` > 0 → 振幅仍高於正常水準
- 門檻：觀察 80th / 20th 百分位數（基於基準期分布）
- 觀察時點：T+1, T+5, T+20 個交易日

**持續性積分分數（Persistence Score）**：

```
積分分數 = Σ 異常振幅(T+k), k = 1 to 20
```

- 積分分數 > 所有事件積分第80百分位數 → 標記為「高持續性事件（High-Persistence Event）」
- 輔助標記：若 `高低價差%(T+1) ≥ 高低價差%(T)` 且 `高低價差%(T+20) < 高低價差%(T+1)` → 標記為「完整弧形事件（Hump Event）」

**進場邏輯**：以 **T 日收盤價買進**，**T+h 日收盤價賣出**。報酬 = 持有 T+1 至 T+h 共 h 日之累積報酬。

**後續報酬統計**（觀察 T+1, T+5, T+20）：

- 累計報酬率分布（histogram + 統計摘要）
- 超額報酬率：(1+R_s)/(1+R_m)−1（個股與大盤累積報酬比值）
- 勝率（累計報酬 > 0 的比例）
- 最大回撤（Maximum Drawdown）
- 平均值、中位數、標準差、偏態、峰態

**後續成交量分析（異常成交量，方案 B）**：

```
異常成交量(T+k) = 成交量(千股)(T+k) / mean(成交量(千股), T-20 to T-1) - 1
```

- 持續性積分分數：Σ 異常成交量(T+k), k = 1 to 20
- 同樣標記「高持續性事件」（積分 > 第80百分位）

**特徵分析**（比較振幅事件中不同特徵組的差異）：

| 特徵變數 | 欄位 |
|---|---|
| 市值 | `市值(百萬元)` |
| 本益比 | `本益比-TSE` |
| 股淨比 | `股價淨值比-TSE` |
| 現金殖利率 | `現金股利率` |
| CAPM Beta | `CAPM_Beta 一年` |

按各特徵的中位數分為高/低兩組，比較兩組的後續累計報酬分布。

## 6. 核心計算公式

### 6.1 振幅（高低價差%）

```
振幅 = (當日最高價 − 當日最低價) ÷ 前日收盤價 × 100%
```

> 直接使用 TEJ 股價資料庫的 `高低價差%` 欄，不重新計算。

### 6.2 異常振幅（方案 B）

```
基準期：事件日前 20 個交易日（不含事件日）
基準期均值 = mean(高低價差%, T-20 to T-1)

異常振幅(T+k) = 高低價差%(T+k) / 基準期均值 - 1
```

### 6.3 異常成交量（方案 B）

```
基準期均量 = mean(成交量(千股), T-20 to T-1)

異常成交量(T+k) = 成交量(千股)(T+k) / 基準期均量 - 1
```

### 6.4 持續性積分分數（Persistence Score）

```
積分分數（振幅）= Σ 異常振幅(T+k), k = 1 to 20
積分分數（成交量）= Σ 異常成交量(T+k), k = 1 to 20

高持續性標記門檻：積分分數 > 所有事件積分的第 80 百分位數
弧形事件（Hump）：高低價差%(T+1) ≥ 高低價差%(T) AND 高低價差%(T+20) < 高低價差%(T+1)
```

### 6.5 累計報酬率

```
T+k 累計報酬率 = Π (1 + 報酬率%(T+j)/100), j = 1 to k  − 1
（或直接加總 報酬率-Ln 欄的對數報酬，再轉換回百分比）
```

### 6.6 累計超額報酬率

```
T+k 累計超額報酬率 = (1 + R_s) / (1 + R_m) − 1
```

其中：
- R_s = 個股 T+1 至 T+k 累積報酬（Π(1+r_s)−1）
- R_m = 大盤（Y9999 加權指數）T+1 至 T+k 累積報酬

> 正確公式可避免加總近似法產生超出 [-100%, ∞) 的不合理數值。

### 6.7 Rolling 百分位數（振幅事件門檻）

```
振幅大事件：今日高低價差% > quantile(高低價差%, q=0.90, window=前20日)
振幅小事件：今日高低價差% < quantile(高低價差%, q=0.10, window=前20日)

異常振幅 > 基準期 80th percentile → 仍偏高（after event）
異常振幅 < 基準期 20th percentile → 已偏低（mean reversion）
```

### 6.8 指數化報酬（多股比較用）

```
指數化報酬(T) = 100 × Π (1 + 報酬率%(t)/100), t = 起始日 to T
```
